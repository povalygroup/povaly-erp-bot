📝 ISSUE ENTRY FORMAT (MANDATORY)

[TICKET] #POV260501 (existing task ticket - required)
[ISSUETICKET] #POV260501-I1 (optional - auto-generated if not provided)
[ISSUE] Short title (required)
[DETAILS] Clear explanation (required)
[PRIORITY] LOW / MEDIUM / HIGH / CRITICAL (required)
[ASSIGNEE] @username (optional)

Example 1 (with ISSUETICKET):
[TICKET] POV260501
[ISSUETICKET] POV260501-I1
[ISSUE] Database connection timeout
[DETAILS] Getting timeout errors when querying user data
[PRIORITY] HIGH
[ASSIGNEE] @tech.lead

Example 2 (without ISSUETICKET - will be auto-generated):
[TICKET] POV260501
[ISSUE] API response delay
[DETAILS] API taking more than 5 seconds to respond
[PRIORITY] MEDIUM

🔄 REACTION-BASED SYSTEM

👍 = CLAIM (user is handling issue)
❤️ = RESOLVED (final completion)
👎 = INVALID / WRONG ISSUE

Rules:
• First 👍 = primary handler
• Multiple 👍 allowed (collaboration)
• ❤️ marks final resolution
• 👎 marks as invalid/wrong issue
• Only reactions control status

═══════════════════════════════════════════════════════════════════

🤖 AUTOMATION LOGIC

On 👍 (Claim):
→ Store user as handler
→ Mark as IN_PROGRESS
→ Send DM to issue creator (auto-delete 60s)
→ Send DM to task assignee if different (auto-delete 60s)

On ❤️ (Resolved):
→ Mark as RESOLVED
→ Lock issue state
→ Send DM to creator (auto-delete 60s)

On 👎 (Invalid):
→ Mark as INVALID
→ Send DM to creator (auto-delete 60s)

═══════════════════════════════════════════════════════════════════

↩️ REACTION REVERSAL SYSTEM

Remove 👍 (Unclaim Issue):
→ Reverts issue to OPEN status
→ Sends unclaim DM to creator (auto-delete 60s)
→ Another user can claim it
→ Smart detection: No unclaim if ❤️/👎 added (state handled by new reaction)

Remove ❤️ (Unresolve Issue):
→ Reverts issue from RESOLVED to IN_PROGRESS
→ Sends unresolve DM to creator (auto-delete 60s)
→ Issue can be re-evaluated

Remove 👎 (Unreject Issue):
→ Reverts issue from INVALID to IN_PROGRESS
→ Sends unreject DM to creator (auto-delete 60s)
→ Issue can be re-evaluated

⚠️ Smart Detection:
• If 👍 removed but ❤️/👎 added: No unclaim notification (it's resolution/rejection)
• All reversal DMs use plain text to avoid Markdown errors
• Reversals allow flexible issue management

═══════════════════════════════════════════════════════════════════

⏱️ ESCALATION (AUTOMATIC)

If not resolved within 2 hours:
→ Alert sent to Admin Control Panel
→ Shows ticket, status, assignee, time elapsed
→ Escalation DM to admins (auto-delete 120s)

═══════════════════════════════════════════════════════════════════

⏰ AUTO REMINDER

If claimed but not resolved within 2 hours:
→ Reminder DM to handler
→ Repeats every 1 hour
