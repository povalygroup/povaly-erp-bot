📌 CORE OPERATIONS — ISSUE RESOLUTION HUB

🎯 PURPOSE: Structured issue-resolution hub for execution problems, technical debugging, and operational blockers.

🔐 ACCESS CONTROL
👑 Authorized: Chairman, CEO, Managers, Assigned Team
👤 All Members: Can use
🚫 No casual chat allowed

📝 ISSUE ENTRY FORMAT (MANDATORY)

[TICKET] #POV260501 (existing task ticket - required)
[ISSUETICKET] #POV260501-I1 (optional - auto-generated if not provided)
[ISSUE] Short title (required)
[DETAILS] Clear explanation (required)
[PRIORITY] LOW / MEDIUM / HIGH / CRITICAL (required)
[ASSIGNEE] @username (optional)

Example 1 (with ISSUETICKET):
[TICKET] POV260501
[ISSUETICKET] POV260501-I1
[ISSUE] Database connection timeout
[DETAILS] Getting timeout errors when querying user data
[PRIORITY] HIGH
[ASSIGNEE] @tech.lead

Example 2 (without ISSUETICKET - will be auto-generated):
[TICKET] POV260501
[ISSUE] API response delay
[DETAILS] API taking more than 5 seconds to respond
[PRIORITY] MEDIUM

🔄 REACTION-BASED SYSTEM

👍 = CLAIM (user is handling issue)
❤️ = RESOLVED (final completion)
👎 = INVALID / WRONG ISSUE

Rules:
• First 👍 = primary handler
• Multiple 👍 allowed (collaboration)
• ❤️ marks final resolution
• Only reactions control status

🤖 AUTOMATION LOGIC

On 👍 (Claim):
→ Store user as handler
→ Mark as IN_PROGRESS
→ Send DM to handler

On ❤️ (Resolved):
→ Mark as RESOLVED
→ Lock issue state
→ Send DM to creator

On 👎 (Invalid):
→ Mark as INVALID
→ Send DM to creator

⏱️ ESCALATION (AUTOMATIC)

If not resolved within 2 hours:
→ Alert sent to Admin Control Panel
→ Shows ticket, status, assignee, time elapsed

⏰ AUTO REMINDER

If claimed but not resolved within 2 hours:
→ Reminder DM to handler
→ Repeats every 1 hour

⚡ COMMANDS

/myissues - View your claimed issues
/openissues - View all unresolved issues
/issue #TICKET - View full details
/close #TICKET - Mark resolved
/unresolved - View claimed but unresolved
/inactive - View unclaimed issues

💬 DISCUSSION SYSTEM

• All replies under original message
• No new messages for same issue
• Thread = single source of truth

⚠️ TOPIC RULES

✓ DO: Post issues, use format, include details, reply in thread
✗ DON'T: Post non-issues, skip fields, create duplicates

🚫 WHAT DOESN'T BELONG HERE

❌ QA submissions → QA Review
❌ Asset storage → Brand Repository
