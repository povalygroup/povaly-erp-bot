🔄 REACTION-BASED QA SYSTEM

👍 = REVIEW STARTED (QA reviewer begins review)
❤️ = APPROVED (QA reviewer approves)
👎 = REJECTED (QA reviewer rejects)

═══════════════════════════════════════════════════════════════════

🤖 AUTOMATION LOGIC

On 👍 (Review Started):
→ Mark as IN_REVIEW
→ Send DM to submitter (auto-delete 60s)
→ Reviewer claims the QA submission

On ❤️ (Approved):
→ Mark QA as APPROVED
→ Mark task as APPROVED (NOT COMPLETED)
→ Send approval DM (auto-delete 90s)
→ Assignee must react ❤️ on Task Allocation to mark COMPLETED

On 👎 (Rejected):
→ Mark QA as REJECTED
→ Mark task as REJECTED
→ Send rejection DM (auto-delete 90s)
→ Task returns to REJECTED state for fixes

═══════════════════════════════════════════════════════════════════

↩️ REACTION REVERSAL SYSTEM

Remove 👍 (Unclaim Review):
→ If PENDING/IN_REVIEW: Reverts to PENDING
→ Sends unclaim DM to submitter (auto-delete 60s)
→ Another reviewer can claim it

Remove ❤️ (Revert Approval):
→ Reverts QA from APPROVED to IN_REVIEW
→ Reverts task from APPROVED to QA_SUBMITTED
→ Sends revert DM to submitter (auto-delete 90s)
→ Reviewer can re-evaluate submission

Remove 👎 (Revert Rejection):
→ Reverts QA from REJECTED to IN_REVIEW
→ Reverts task from REJECTED to QA_SUBMITTED
→ Sends revert DM to submitter (auto-delete 90s)
→ Reviewer can re-evaluate submission

⚠️ Smart Detection:
• If 👍 removed but ❤️/👎 added: No unclaim notification (it's approval/rejection)
• All reversal DMs use plain text to avoid Markdown errors
• Reversals preserve original reviewer info for tracking

👷 FOR EMPLOYEES

WORKFLOW:
1. Complete task in Task Allocation
2. React 👍 to mark STARTED
3. Complete your work
4. Post submission in QA & Review topic
5. Wait for review (get DM when reviewer claims it)
6. If ❤️: Task APPROVED! React ❤️ on Task Allocation to mark COMPLETED
7. If 👎: Fix issues, resubmit

⚠️ Important:
• Submit only when 100% complete
• Include all deliverables
• Provide clear submission link
• Add helpful notes
• Check DMs for status updates
• After QA approval, confirm completion on Task Allocation

🔔 DM Notifications (All Auto-Delete):
• Review started: 60 seconds
• QA approved: 90 seconds
• QA rejected: 90 seconds
• Review unclaimed: 60 seconds
• Approval/rejection reverted: 90 seconds

═══════════════════════════════════════════════════════════════════

👑 FOR QA REVIEWERS

WORKFLOW:
1. Check /pendingqa or /unreviewedqa for submissions
2. Review oldest first
3. React 👍 to claim and mark review started
4. Check deliverables vs requirements
5. React ❤️ to approve OR 👎 to reject
6. If rejecting, use /reject command with specific reason

⚠️ Important:
• Review within 2 hours
• Provide clear rejection reasons
• Be specific about requirements
• Approve only if 100% meets specs
• Can revert decisions by removing reactions

🔄 Reversal Options:
• Remove ❤️: Revert approval, re-evaluate
• Remove 👎: Revert rejection, re-evaluate
• Remove 👍: Unclaim review, let others review
